How it works

Cortex extends your agent through the Model Context Protocol (MCP). Each component is an MCP server that plugs into Claude Code / Cursor / Windsurf. ┌──────────────────────────────────────────────────────────┐ │ CORTEX (plug-in layer) │ │ │ │ Dispatcher A2A Hub Forge Vibe Replay │ │ remote cmd agent comms tool gen session mem │ │ │ │ MCP MCP MCP MCP + Hooks │ └──────────────────────────┬───────────────────────────────┘ │ ┌────────────▼─────────────────┐ │ Your Agent │ │ Claude Code / Cursor / etc. │ └──────────────────────────────┘

Components

Dispatcher

Beta

Control your agent from your phone via Telegram. Session management, project routing, concurrent tasks.

pip install agent-dispatcher

A2A Hub

Alpha

Agent-to-Agent protocol. WebSocket hub with service discovery, task delegation, and MCP bridge.

pip install a2a-hub

Forge

Alpha

Self-evolving tool agent. Describe what you need — Forge generates, tests, iterates, and installs it.

pip install forge-agent

Vibe Replay

Alpha

Capture AI coding sessions, extract decisions and patterns, generate shareable HTML replays.

pip install vibe-replay

Install

$ pip install cortex-cli-agent[all]
$ cortex init
$ cortex status
