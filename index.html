Components

Built to work alone. Better together.

Each component is a standalone, open-source project. Use one, use all four. They're designed to compose naturally.

D

Dispatcher

Command Layer

Your mobile JARVIS. Message Claude from your phone via Telegram. It classifies, routes, and manages concurrent tasks — always on, always ready.

  • Smart message classification (task / status / cancel)
  • Project-aware routing with auto directory detection
  • Concurrent task management (3 parallel sessions)
  • Intelligent follow-up detection (no explicit reply needed)
  • Progress updates and typing indicators
  • 570 lines, zero dependencies, LaunchAgent daemon
H

A2A Hub

Communication Layer

Agent-to-Agent protocol hub. Agents discover each other, delegate tasks, and collaborate through a lightweight WebSocket-based protocol.

  • Agent registry with capability-based discovery
  • Task delegation with routing and result collection
  • Broadcast messaging for multi-agent coordination
  • Heartbeat and automatic disconnection cleanup
  • MCP bridge — use directly from Claude Code
  • Python SDK with decorator-based agent creation
F

Forge

Creation Layer

Self-evolving tool agent. Describe what you need in plain English — Forge builds it, tests it, and installs it into your toolkit.

  • Natural language to working tool in one command
  • Intelligent clarification questions for ambiguous requests
  • Auto-generated tests with iterative refinement (max 5 rounds)
  • Multiple output types: MCP tool, CLI command, Python module
  • Direct installation into MCP config or system PATH
  • Both CLI and MCP server interfaces
V

Vibe Replay

Memory Layer

Not just logs — structured wisdom. Captures coding sessions, extracts decisions and learnings, generates beautiful shareable replays.

  • Claude Code hooks for automatic event capture
  • Phase detection (exploration, implementation, debugging)
  • Decision point and turning point identification
  • Auto-extracted reflections, learnings, and patterns
  • Interactive HTML replay with timeline and code diffs
  • Cross-session wisdom aggregation
How It Works

From thought to execution to wisdom

A real workflow: you're on your phone, you have an idea, and your agent infrastructure handles the rest.

01

You send a message from your phone

"Refactor the data pipeline in sims, add proper error handling and tests."
Sent via Telegram. You're on the subway. No laptop needed.

02

Dispatcher classifies and routes

Detects project sims, sets working directory, classifies as engineering task. Spawns a Claude Code session with 50 max turns and your memory context. Sends you: "On it (sims)."

03

Agent discovers it needs a CSV parser tool

During refactoring, Claude realizes it needs a custom CSV parser. Through A2A Hub, it finds Forge and delegates: "Build a SIMS sensor CSV parser with timestamp filtering."

04

Forge builds, tests, and installs the tool

Forge generates the parser, creates 12 tests, runs them, all pass. Installs as an MCP tool. The refactoring agent now uses it directly.

05

Task completes. Results sent to your phone

"Done. Refactored data_loader into 3 modules, added error handling and 24 tests. Also created a new sensor_csv_parser tool (installed to MCP)."

06

Vibe Replay captures everything

The entire session is captured: 4 key decisions (why 3 modules instead of 2), 2 learnings (SIMS timestamp format quirks), 1 new tool created. HTML replay generated. Wisdom aggregated.

Quick Start

Up and running in minutes

Each component installs independently. Start with what you need.

Dispatcher — Mobile JARVIS Python 3.11+
# Clone and configure
git clone https://github.com/zzhiyuann/dispatcher
cp .env.example .env # Add your Telegram bot token & chat ID

# Install as always-on daemon (macOS)
python3 dispatcher.py --install-daemon

# Or run directly
python3 dispatcher.py
✓ Dispatcher running. Send a message on Telegram.
A2A Hub — Agent Communication Python 3.11+ · pip
# Install
pip install a2a-hub

# Start the hub server
a2a-hub start # WebSocket on port 8765

# Add MCP bridge to Claude Code
a2a-hub bridge # Starts MCP server for Claude Code
✓ Hub running. 0 agents connected.
Forge — Self-Evolving Tool Agent Python 3.11+ · pip
# Install
pip install forge-agent

# Create a tool from natural language
forge create "convert CSV to JSON with column filtering"
✓ Generated · 8 tests passed · Installed to ~/.forge/tools/

# Or install directly as MCP tool
forge create "parse URL query parameters" --type mcp --install mcp
✓ Added to ~/.mcp.json. Available in Claude Code now.
Vibe Replay — Session Wisdom Python 3.11+ · pip
# Install
pip install vibe-replay

# Hook into Claude Code (one-time setup)
vibe-replay install # Adds hooks to ~/.claude/settings.json

# After coding sessions, view your replays
vibe-replay sessions # List captured sessions
vibe-replay replay latest # Open interactive HTML replay
vibe-replay wisdom # See aggregated learnings across sessions
✓ 3 decisions · 2 learnings · 1 pattern detected
